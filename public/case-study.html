Background

Where it started

My role in the original project

UX/UI Designer & Style Guide Contributor

I joined the Homefield Marketing project in its early phase as one of the UX/UI designers involved in shaping the visual foundation of the website. My contribution focused on establishing the design system that would guide the entire build: the photographic treatment and mood direction, icon set style, colour palette, typographic hierarchy, spacing scale, and the responsive layout structure.

The style guide was then handed off to another talented designer who took those foundations and completed the full visual design of the site. The result was a cohesive set of polished comps ready for development.

Why I rebuilt it

Closing the gap between design intent and delivered product

The original production site, while functional, had moments where the design guidelines weren't fully translated into code. This is a natural tension in any collaborative web project — timelines tighten, priorities shift, and small design details can be deprioritised during development.

I have a great deal of respect for everyone who contributed to the original build. This personal renewal is not a critique of anyone's work — it is simply my own exploration of what the site might look like with more time dedicated to bridging that gap between the design system and the final implementation.

It also gave me the opportunity to modernise the architecture: replacing WordPress/ACF with Next.js 15, adding ISR via Supabase, and layering in proper performance, SEO, and accessibility practices.

Design System

Style guide contributions

The following areas represent my direct contributions to the Homefield Marketing design system before visual design was completed by the team.

Colour Palette

Defined the core two-tone brand palette — deep navy #101820 as the anchor and heritage gold #DAAA00 as the accent — along with supporting neutrals and their usage rules.

Typography System

Selected and specified the typeface trio: Superior Title for display headings, Montserrat for UI elements and navigation, and Open Sans for body copy. Defined the full size scale, weight pairings, and leading values.

Layout & Spacing

Established the 8px baseline grid, section padding rhythm, column gutters, and the max-width container model (1440px) with consistent horizontal padding across all breakpoints.

Icon Style Direction

Defined the visual style for UI icons: filled, rounded, 24px grid, aligned to the brand's approachable-yet-professional tone. Established the usage rule of gold fill on dark backgrounds.

Photo & Visual Mood

Directed the photographic concept: authentic prairie landscapes, warm natural tones, a dark overlay treatment to unify varied source photography, and an emphasis on community and local business imagery.

Responsive Breakpoints

Defined the three-tier breakpoint system (mobile / tablet / desktop) and key behaviour rules: how the header collapses, how the grid reflows, and how type scales down gracefully on smaller viewports.

The Challenge

Design intent in practice

The gap between a well-crafted design system and its production implementation is a challenge in virtually every web project. This one was no exception.

"A design guide is only as strong as the care taken to follow it through to every element of the final build."

The original Homefield Marketing website was built under real-world constraints — tight timelines and shifting priorities that are common in agency production. The talented people involved did their best work within those constraints, and the result was a functional, live website serving real clients.

In the finished product, however, there were moments where the spacing rhythm felt compressed, where font weights didn't match the specified hierarchy, where hover states and transitions were simplified, and where the responsive breakpoint behaviour drifted from the original intent.

This renewal project gave me the space — without deadline pressure — to revisit every one of those details and implement them exactly as the style guide intended.

Architecture

Technology choices

Moving from WordPress/ACF to a modern static-site-first stack let me prioritise performance, maintainability, and developer experience.

Framework

Next.js 15

App Router, SSG, ISR, and server components — the right tool for a content-driven marketing site.

Language

TypeScript

Strict typing across data models, component props, and API responses eliminates entire classes of bugs.

Styles

Tailwind v4 + SCSS

Tailwind for layout utilities, SCSS/BEM for component-scoped styles that mirror the design system naming.

Animation

GSAP 3

ScrollTrigger-powered text reveals and staggered entrances — matching the premium feel of the original design.

Database

Supabase

PostgreSQL-backed project data with ISR revalidation. Static pages still serve from cache if the DB is paused.

Hosting

Vercel

Edge-deployed globally, seamless GitHub integration, preview deployments per pull request.

Development

Technical highlights

Beyond replicating the visual design, the renewal included deliberate engineering decisions around performance, accessibility, and SEO.

Performance — lazy-loaded below-fold components

The Hero component loads as a static import (above the fold, no delay). All remaining homepage sections — IntroText, FeaturedWork, MarketingSolutions, ContactForm — are dynamically imported with next/dynamic, splitting GSAP and other heavy code out of the initial JS bundle. The hero video uses preload="metadata" to prevent unnecessary network load on page entry. Images are served as AVIF/WebP via Next.js Image Optimization.

SEO — per-page metadata, OG image, sitemap, structured data

Each page exports a generateMetadata function with a canonical URL, Open Graph tags, and Twitter card data. A dynamic OG image is generated at build time using next/og ImageResponse — no static asset required. sitemap.ts outputs a full XML sitemap covering all static and dynamic routes. A JSON-LD MarketingAgency schema is embedded on the homepage for rich-result eligibility.

Accessibility — WCAG 2.1 AA baseline

Skip navigation link (visually hidden, visible on focus). Semantic HTML landmarks throughout: role="banner", named <nav> elements for both desktop and mobile overlays, role="search" on search forms. The decorative hero video is wrapped in aria-hidden="true". Form submission state is announced to screen readers via aria-live="polite". Focus rings use :focus-visible so keyboard users see clear indicators without mouse-click flicker.

Header — pixel-accurate via JS computed-style inspection

The Solutions mega-dropdown, gold underline animation, and mobile nav overlay were validated against the original site using getComputedStyle measurements in DevTools. Key values confirmed this way: header height (112px desktop / 82px mobile), nav font (Montserrat 700 14px), gold underline width transition (8px → 48px over 400ms), dropdown close debounce (80ms timer), and CTA button height (31px, 11px text).

ISR + Supabase — resilient static delivery

Project detail pages are pre-rendered at build time via generateStaticParams pulling slugs from Supabase. revalidate: 3600 attempts hourly background regeneration. Crucially, if Supabase is paused (as it is on the free tier after inactivity), Vercel continues to serve the last-built static pages — the site remains fully functional without any database connection.

Reflection

What I took away

This project reinforced how much value sits in the handoff — and how rewarding it is to own every step from system design to production code.

01

Design systems need champions in the build

A style guide doesn't enforce itself. Having the designer involved in — or directly doing — the front-end implementation is the most reliable way to preserve design intent across a full production build.

02

Modern tooling makes precision more accessible

Next.js, Tailwind, and TypeScript together made it straightforward to translate spacing tokens, colour values, and typographic scales directly into code — with the type system catching inconsistencies at compile time.

03

Performance and accessibility are design decisions

The best visual work deserves to be experienced at full quality — which means fast load times, keyboard accessibility, and screen-reader support are just as important as pixel-perfect spacing and the right font weight.
